## How to Use

### For interview prep (1 week before)
1. Read the **"30-Second Answer"** of each question — flag the ones you can't answer
2. For flagged ones, read the full **"Deep Dive"**
3. Memorize the **"Red Flag If You Say"** anti-patterns

### For interview prep (1 day before)
1. Read only the **TOP 10 across all files** (marked ⭐)
2. Re-read **`10-Common-Gotchas-Bugs.md`** in full — these come up in EVERY senior interview

### In the interview
- If asked about a code-level pattern → use the **production scenario files** (`/Java/02..06`)
- If asked about a Spring concept → use the **Q&A files here**

## Format of Each Question

Each question follows the same structure:

```
### Q##: The question (⭐ if top-10)
**30-Second Answer:** Crisp 2-3 sentence answer for the interview.

**Deep Dive:** Full technical explanation with code.

**Common Follow-Ups:**
- Related question 1
- Related question 2

**Red Flag If You Say:**
- Wrong answer pattern that signals you don't really know
```
